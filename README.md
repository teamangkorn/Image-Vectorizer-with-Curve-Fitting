# Image Vectorizer with Curve Fitting

A self-contained C++17 raster-to-vector image converter built for the
**Computational Geometry** final project (term 2/2568, Chulalongkorn CE).

The pipeline composes three classical algorithms from the course and
is benchmarked against two simpler **baselines** that share the same
front-end but stop earlier — so we can quantify how much curve fitting
actually buys us.

## Problem

Convert a raster image (PNG / JPG / BMP) into a Scalable Vector
Graphics (SVG) file whose paths approximate the original boundary to
within a user-supplied tolerance, while being as compact as possible.
Vectorization is the inverse of rasterization, is ill-posed in
general, and is solved here as a chain of three geometric
approximation steps.

## Approach

- **Baseline-0 (`raw`)** — emit the raw pixel-outline polygon
  produced by Moore-Neighbour boundary tracing, with no
  simplification and no curve fitting.
- **Baseline-1 (`polyline`)** — Moore-Neighbour + Ramer–Douglas–Peucker
  (RDP) polyline simplification, then emit polygons.
- **Proposed (`bezier`)** — Moore-Neighbour + RDP +
  **Schneider's** recursive cubic-Bézier fit, evaluated throughout
  with the **de Casteljau** subdivision scheme.

Mode is selected on the command line with `-m raw|polyline|bezier`.

## Results

Median over 5 runs, scaling experiment at $n \in \{10^3, 10^4, 10^5\}$
raw boundary vertices (Apple M2, macOS Sequoia 15.4, Apple Clang 17, `-O3`):

| n     | bytes (raw) | bytes (polyline) | bytes (proposed) | size reduction | runtime speedup |
|------:|------------:|-----------------:|-----------------:|---------------:|----------------:|
| 1k    |      16,783 |            1,053 |          **619** |      **27.1×** |          1.36×  |
| 10k   |     172,522 |           15,146 |        **8,082** |      **21.3×** |          1.25×  |
| 100k  |   1,754,478 |          144,798 |       **84,630** |      **20.7×** |          1.05×  |

The proposed pipeline is **20–27 × smaller** than the raw baseline and
**1.7–1.8 × smaller** than the RDP-only baseline, while remaining the
**fastest** end-to-end at every tested scale (curve fitting adds < 1 ms
even at n = 100 k, and the smaller output saves more
SVG-serialisation time than fitting costs).  See `report/report.pdf`
§VII for the full per-stage breakdown and
`experiments/results/{runtime,size}_vs_n.png` for the log-log plots.

Dataset: synthetic concentric annuli generated by
`build/gen_synthetic` so the boundary length is exactly controllable
without inflating the raster (see [data/synthetic/README.md](data/synthetic/README.md)).

## How to Run

Requires CMake ≥ 3.15 and any C++17 compiler.

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j
```

Or use the convenience targets:

```bash
make demo    # mandatory CLI demo (input -> process -> result + perf)
make bench   # scaling experiment, writes experiments/results/runtime.csv
make plot    # render runtime_vs_n.png and size_vs_n.png (needs matplotlib)
make test    # smoke tests for all three modes
```

### Direct CLI usage

```
vectorizer <input-image> [options]
```

| Option                     | Default     | Meaning                                                                |
|----------------------------|-------------|------------------------------------------------------------------------|
| `-o, --output <file>`      | `out.svg`   | Output SVG path                                                        |
| `-m, --mode <name>`        | `bezier`    | `raw` (baseline-0), `polyline` (baseline-1), or `bezier` (proposed)    |
| `-t, --threshold <int>`    | `128`       | Luminance threshold in `0..255` for binarization                       |
| `--invert`                 | off         | Treat lighter pixels as foreground                                     |
| `-e, --epsilon <float>`    | `1.5`       | RDP tolerance, in pixels                                               |
| `-b, --bezier-error <f>`   | `2.0`       | Maximum Bézier fitting error, in pixels                                |
| `--no-curves`              | off         | Alias for `--mode polyline`                                            |
| `--min-area <int>`         | `4`         | Drop components smaller than N pixels                                  |
| `--fill <color>`           | `#000000`   | SVG fill color                                                         |
| `--bench`                  | off         | Print one CSV-style benchmark line on stdout (used by `make bench`)    |
| `--quiet`                  | off         | Suppress per-stage progress on stderr                                  |
| `-h, --help`               |             | Show usage                                                             |

### Example

```bash
./build/vectorizer logo.png -o logo.svg -e 1.0 -b 1.0
./build/vectorizer logo.png --mode raw      -o logo_raw.svg
./build/vectorizer logo.png --mode polyline -o logo_poly.svg -e 1.0
```

## Project Layout

```
final_project/
├── CMakeLists.txt
├── Makefile                       # demo / bench / plot / test shortcuts
├── README.md
├── src/                           # C++17 source
│   ├── main.cpp                   # CLI driver (mode + bench instrumentation)
│   ├── image.{hpp,cpp}            # PNG load + binarization (stb_image)
│   ├── tracer.{hpp,cpp}           # connected components + Moore-Neighbour
│   ├── rdp.{hpp,cpp}              # Ramer-Douglas-Peucker
│   ├── bezier.{hpp,cpp}           # de Casteljau + Schneider fit
│   ├── svg.{hpp,cpp}              # SVG path serializer
│   ├── geometry.hpp               # shared structs (Point, Polyline, ...)
│   ├── stb_image_impl.cpp         # one-line stb_image implementation
│   └── gen_synthetic.cpp          # PGM generator for the scaling experiment
├── third_party/stb_image.h        # public-domain PNG/JPG loader
├── data/{synthetic,real}/         # test images (synthetic generated on demand)
├── experiments/
│   ├── benchmark.sh               # scaling experiment driver
│   └── results/runtime.csv        # raw measurements + plots
├── visualization/plot_results.py  # matplotlib plotter for runtime.csv
├── demo/                          # mandatory demo (Project Guideline §8, type C)
├── tests/test_algorithms.sh       # smoke tests for all three modes
├── report/{report.tex,report.pdf} # LaTeX two-column technical report
└── presentation/presentation.txt  # slide-by-slide script
```

## Reproducing the Numbers in the Report

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release && cmake --build build -j
make bench       # writes experiments/results/runtime.csv
make plot        # writes runtime_vs_n.png, size_vs_n.png
make test        # asserts pipeline correctness
cd report && pdflatex report.tex && pdflatex report.tex
```

- R. T. Farouki, *The Bernstein polynomial basis: A centennial retrospective*, CAGD 29(6), 2012.
- R. C. Gonzalez, R. E. Woods, *Digital Image Processing* (4e), §11.1 (boundary tracing).
